# Informe de Avance 14-08-2025

## Resumen

Este informe detalla el progreso realizado en el desarrollo del módulo `industry_real_estate` de Odoo, basado en los avances realizados en la semana del 10 al 14 de agosto.

## Funcionalidades Implementadas

- **Firmas y documentos + propiedades:**
    - Se realizaron correcciones en la visual de las direcciones de la propiedad
    - Se realizaron correccines en la visual de la geolocalización
    - Se implemento funcionalidad base del botón de geolocalización
    - Se realizaron correcciones de desarrollos previos para estabilizar comunicaciones entre modulos, esto haciendo referencia a firmas y contratos de arrendamiento
    - Se realizó la implementación de las firmas y documentos al modulo ECOERP.
    - Se realizó la creación de vistas y funcionalidad base para el estado "contrato firmado"
    - Se dejó la base para la implementación de la funcionalida de firmas a los demás estados del contrato

- **NOTAS**
    - Las plantillas se deben crear con condicciones restrictivas para asociarlas al contrato, ejemplo: Si el contrato tiene numeración S0001, la plantilla del contrato debe tener el nombre "contrato_de_arrendamiento_S0001".
    - Esto es solo por si se desea tener un control contra errores de usuarios a nivel administrativo y poder validar cada uno de los estados con un documento específico según corresponda.
    - Para asociar un documento firmado o crear la firma para validar el estado, se debe tener creado previamente las plantillas. Esto debido a que el modal no permite adjuntar documentos con la opción de creación rápida que ofrece Odoo.
- **Pendiente**
    - Modificacion en vistas para anexar las cantidades recepcionadas queda pendiente a validacion
    - Duplicidad de registros al crear un nuevo moviento de inventario o historial.
    - Anexar la funcionalidad de las firmas a los demás estados.
    - Culminar funcionalidad del botón de geolocalización para que el mapa tenga el acercamiento correspondiente haciendo uso de StreetMaps
    - Completar la funcionalidad el otro sí, si los parágrafos y clásulas ya fueron terminadas.
    - Agregar la ubicación a cada item (producto).
    - Agregar el usuario faltante como responsable al momento de hacer la recepción de un producto, el cual sería "Otros" y permitir registrar a mano esa responsable.
